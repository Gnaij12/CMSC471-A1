Temperature Option:

Write Up

  1. Dataset: We used the Daily Weather in the U.S., 2017 Dataset. This dataset contains weather data from all of the United States, including the territories. It contains temperatures, winds, and precipitation data. However, we decided to take a subset of the data by only using the 50 states and only using temperature from those states. We use minimum temperature, maximum temperature, and our own temperature range variable to display. We also use the date, the state, and the longitude/latitude for making our visualization.

  2. Rationale for our design decisions: Because our weather data has longitude and latitude coordinates, we thought it would be more interesting and understandable to use a map. Therefore, we displayed a US map and mapped the coordinates onto that. Then, we wanted a way to easily show temperature differences, so we used both size and color to do so. However, with the natural differences in temperatures accross different months and days, we wanted to allow users to check specific days with a slider. We figured the slider would be the easiest and most intuitive method to do this. For both temperature minimums and maximums, we found that both color and size were appropriate embeddings for the values. Temperatures have extremes on both hot and cold ends, a size scaling that initially shrinks, then decreases, is able to capture magnitudes on both end of the spectrum. Color is commonly used to represent tempreature, and so we developed a scale that uses darker blues for cold temperatures and darker reds for hot temperatures, with lighter colors for more moderate temperatures. For temperature ranges however, we decided not to include the color embedding, since ranges do not necessarily have a 'hotness' or 'coldness' associated with them. Instead, we allowed the size embedding to demonstrate the magnitude of the range. To select which temperature variable to show, the most intuitive option was to use a dropdown.
